### Next.js + Django 연동작업 

## 1. 전체 작업 개요
Nextjs 프론트 + Django 백엔드 연동 이후 
프로젝트 카드/모달/디자인 톤을 본격적으로 다듬었다! 
UI/UX 쪽에서 많은(나름..) 디테일을 건드렸다! 

오늘은 큰 이슈는 없었고 자잘한 버그들이 있었다 
(어이없는 것들도 있긴한데 .. ) 정리해보기로 하자!

## 1) Nextjs 화면 오른쪽이 검게 보이는 버그 

- 현상 : 말 그대로 개발 모드에서 화면 오른쪽이 갑자기 까맣게 보임

- 원인 : 찾아본 결과 Nextjs Dev Overlay(HMR)가 화면 계산을 잘못할때 발생하는
        일시적 개발모드 전용 버그라고 한다 

- 해결 : 새로고침/브라우저 줌 초기화로 사라진다고 했지만 여전히 사라지진 않았었고, 
        main class 뒤쪽에 w-full을 작성하고 로컬 서버를 종료하고 다시 실행 후 정상적으로 출력됨!


## 2) 사이드바 정렬 및 스타일 정리 중 있었던 버그(?)

- 현상 : 아이콘 + 텍스트 정렬이 깨짐 -> inline-flex items-center gap-2 적용을 시켰음에도 
        아이콘과 텍스트가 너무 딱 붙어있었음

- 원인 : 오타... gap-2 를 gqp-2 로 작성했었다 ..

- 해결 : gqp를 .. gap으로 수정했더니 정상적으로 배치/정렬됨..



## 2. 최종 결과

- 프론트쪽은 카드/모달/사이드바 UI 완성도가 많이 좋아졌다

- 디자인 톤을 파스텔 블루그레이톤(#DDE6F5)로 채택하고 텍스트 톤도 #222, #333, #555로 재정비
    -> 훨씬 부드럽고 깔끔한 느낌의 UI가 완성됨! 

- github, Mail 아이콘이 튀지 않도록 CSS 필터를 style={{ filter: "grayscale(30%) brightness(90%)" }} 줌
    -> 굳이 CSS 필터를 준 이유는 Tailwind filter는 미세조정이 불가능 혹은 어렵고, inline style이 더 자연스럽고 유연하다 라는 설명을 봤다
